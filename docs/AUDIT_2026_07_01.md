1. `lib/widgets/isle_card.dart` (new, untracked) — uses old `Isle.mass`, calls itself a "floating isle card" — directly contradicts spec § 2 (Isle Spark, no mass).
2. `lib/models/isle.dart` — `mass` field — contradicts spec § 8 (delete mass).
3. `lib/widgets/mountain_visual.dart` — contradicts spec § 8 (delete).
4. `lib/widgets/sparse_lines_background.dart` — contradicts spec § 8 (delete, "contradicts calm-water vision").
5. `lib/widgets/spark_button.dart` (old) — contradicts spec § 9 (`CreateSparkButton` is dashed + `?`, not gradient ✨).
6. `lib/screens/main/main_screen.dart` (modified, uncommitted) — still uses `MountainVisual(mass:…)` + sparse-lines painter + `_calculateTotalMass` — directly contradicts spec § 1 (Home = floating sparks on water).
7. `lib/screens/auth/*.dart` (login + signup, 755 lines combined) — full-on dark theme that contradicts the locked light theme.
8. `lib/main_test.dart` — dark theme test screen, no longer relevant.
9. `lib/providers/isle_provider.dart` — mock data names (`'Fitness Journey'`, etc.) contradict the new spec (a spark's identity is its emoji, not a name).
10. `lib/config/supabase_config.dart` — hardcoded Supabase project URL + publishable key in a public repo (security smell; not in spec but a project-level concern).

## § 5 — Recommended Actions (in Priority Order)

1. **🔒 Rotate the Supabase publishable key** in `lib/config/supabase_config.dart` and replace URL + key with clearly-fake placeholders (or move to `archive/`). It's a public repo.
2. **Decide on the uncommitted Flutter changes** in the working tree (`main_screen.dart` + `isle_card.dart`). The new `IsleCard` is itself vestigial (uses `Isle.mass`); the partial `main_screen.dart` refactor mixed the new `IsleCard` with the old `MountainVisual` + dark theme. **Suggested:** revert both, then re-do the migration against the spec's § 8 model map.
3. **Move fully-vestigial docs to `docs/archive/`**: `UI_DEVELOPMENT_PLAN.md`, `docs/ARCHITECTURE.md`, `docs/DEVELOPMENT.md`, `docs/design/SCREENS.md`. Update `docs/archive/README.md` "What to Read Instead" table to drop these as active pointers, and add `BEACH_LINE.md` to the "What's in Here" table.
4. **Surgically clean `docs/design/VISION.md` and `docs/design/TOKENS.md`**: strip the contradicted sections (mountain metaphor, no-streaks, Layout section, mass-tokens, "✨" spark-button iconography). Keep the Three-Words vibe and the core token system.
5. **Update `README.md`**: rewrite "What This Is" section (no more Isle / Goal / Sub-point), link to `ISLE_SPARKS_SPEC.md` and `MOCKUPS.md` first, drop the stale "Mock data (no backend)" line, refresh the project-structure tree.
6. **Update `CURRENT_STATUS.md`**: bump "Latest commit" from `010eeb0` to `c6181f7`; drop `UI_DEVELOPMENT_PLAN.md` from "Current docs" once archived.
7. **Update `docs/design/README.md` Status checklist** (Phase 1–7 statuses no longer apply — design is locked; what's left is implementation).
8. **Optionally** move `goal_isle_working_mockup.html` to `docs/archive/` (already marked as archive, but file location is misleading).
9. **Code migration** is its own scope (per spec § 8): do all the deletes / renames / adds for `lib/models/`, `lib/providers/`, `lib/screens/`, `lib/widgets/`. Then build the new `IsleSpark` widget, `CreateSparkButton`, `ShapePicker`, and `SparkDetailsScreen` per spec § 9.
